## ⚠️ 安装前检查清单

1. **查看源码**：确认扩展没有记录密钥或执行意外的网络请求
2. **检查权限**：使用 `pi-sandbox` 或 `@gotgenes/pi-permission-system` 限制扩展权限
3. **查看维护状态**：优先选择 GitHub 上有最近提交的包
4. **检查依赖**：`npm info <pkg> dependencies` 确认依赖树健康
5. **使用 pi-pkg-guard**：自动检测未在 PI 设置中注册的 npm 包

## 📊 质量风险提示

1. **重复实现**：同一功能域有多达 5+ 竞争实现（如 memory），建议选择社区活跃度最高的
2. **废弃风险**：月下载量 < 100 的扩展占 60%+，可能被作者放弃
3. **安全注意**：任何 `bash` 权限的扩展都应在沙箱中测试
4. **版本兼容**：PI 核心 API 仍在快速迭代，扩展需要跟进
