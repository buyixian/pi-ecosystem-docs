## 概述

让代理**知道何时该问人**。这不是传统的 UI 组件，而是"代理向上管理"的能力——结构化提问、待办跟踪、第二意见请求。

## 设计理念

rpiv 系列的核心洞察：**代理的瓶颈不是能力，是信息不对称**。

```
传统模式: 代理猜测 → 执行 → 用户纠正
rpiv 模式: 代理识别不确定性 → 结构化提问 → 获得明确答案 → 执行
```
